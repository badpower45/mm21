# نظام نقطة البيع - دليل الاستخدام

## نظرة عامة
نظام POS محلي متكامل مع واجهتين منفصلتين:
- **واجهة الموظف**: بسيطة ومخصصة للبيع فقط
- **واجهة المالك**: شاملة لإدارة كل جوانب النظام

## المستخدمون الافتراضيون

### المالك (Owner)
- اسم المستخدم: `owner`
- كلمة المرور: `123456`
- الصلاحيات: وصول كامل لجميع الميزات

### الموظف (Cashier)
- اسم المستخدم: `cashier` أو `cashier2`
- كلمة المرور: `123456`
- الصلاحيات: شاشة الكاشير فقط

## واجهة المالك - الميزات

### 1. لوحة التحكم (Dashboard)
- إحصائيات المبيعات والأرباح اليومية
- رسوم بيانية للمبيعات الأسبوعية
- تنبيهات المخزون المنخفض
- اقتراحات الشراء التلقائية

### 2. نقطة البيع (POS)
- نفس واجهة الموظف مع إمكانية رؤية التكاليف والأرباح
- إضافة منتجات للسلة
- إتمام عمليات البيع وطباعة الإيصالات

### 3. المنتجات والوصفات (Products & Recipes)
- عرض جميع المنتجات
- إضافة/تعديل المنتجات
- إدارة الوصفات (المكونات والكميات)
- حساب التكلفة التلقائي من الوصفة

### 4. إدارة المخزون (Inventory)
- متابعة المواد الخام
- تعديل الكميات (إضافة/خصم)
- تنبيهات الحد الأدنى
- حساب قيمة المخزون الإجمالية

### 5. التقارير (Reports)
- تقارير المبيعات التفصيلية
- أداء الموظفين
- إحصائيات يومية وشاملة

### 6. إدارة المستخدمين (Users)
- إضافة/تعديل/حذف المستخدمين
- تحديد الصلاحيات (مالك/موظف)
- تفعيل/تعطيل المستخدمين

### 7. الإعدادات (Settings)
- اسم المتجر
- رسالة الشكر في الإيصال
- إعدادات الطابعة
- نسبة الضريبة
- العملة

## واجهة الموظف - الميزات

### شاشة الكاشير
- شبكة منتجات كبيرة وواضحة
- سلة مشتريات بسيطة
- عرض التكاليف والأرباح لكل منتج
- أزرار (+/-) لتعديل الكمية
- زر دفع كبير وواضح
- طباعة إيصال تلقائية

## البيانات التجريبية

### المنتجات
1. **صودا**: 40 ج.م تكلفة، 80 ج.م سعر بيع، 40 ج.م ربح
2. **عصير برتقال**: 35 ج.م تكلفة، 70 ج.م سعر بيع، 35 ج.م ربح
3. **قهوة**: 25 ج.م تكلفة، 55 ج.م سعر بيع، 30 ج.م ربح

### المواد الخام
- صوص، كوب، سيفن، ثلج، شفاطة
- كل مادة لها: كمية حالية، حد أدنى، حد مستهدف، تكلفة وحدة

## الحسابات التلقائية

### تكلفة المنتج
يتم حسابها تلقائياً من مجموع تكلفة مكونات الوصفة:
```
تكلفة المنتج = Σ (كمية المكون × تكلفة وحدة المكون)
```

### الربح
```
الربح = سعر البيع - التكلفة
نسبة الربح = (الربح / سعر البيع) × 100
```

### اقتراحات الشراء
```
الكمية المطلوبة = الحد المستهدف - الكمية الحالية
التكلفة التقديرية = الكمية المطلوبة × تكلفة الوحدة
```

## تدفق العمل النموذجي

### للموظف
1. تسجيل الدخول بحساب الموظف
2. اختيار المنتجات بالضغط على البطاقات
3. مراجعة السلة وتعديل الكميات
4. الضغط على "الدفع"
5. طباعة الإيصال
6. بداية عملية بيع جديدة

### للمالك
1. تسجيل الدخول بحساب المالك
2. مراجعة لوحة التحكم للاطلاع على الأداء
3. التحقق من تنبيهات المخزون
4. إدارة المنتجات والوصفات عند الحاجة
5. مراجعة التقارير لتحليل الأداء
6. إضافة مستخدمين جدد أو تعديل الصلاحيات

## عرض الموبايل (للمالك فقط)

يمكن للمالك التبديل إلى "عرض الموبايل" لمتابعة:
- المبيعات والأرباح اليومية
- المواد المنخفضة
- اقتراحات الشراء
- ملخص سريع للأداء

## ملاحظات مهمة

- جميع البيانات محفوظة في localStorage (محلياً)
- النظام لا يتطلب اتصال بالإنترنت
- عند البيع، يتم خصم المكونات تلقائياً من المخزون
- الإيصالات تستخدم إعدادات المتجر المحفوظة
- التقارير والإحصائيات تحدث فورياً

## تطوير مستقبلي محتمل

- إضافة Supabase للمزامنة بين الأجهزة
- طباعة حرارية مباشرة
- نسخ احتياطية تلقائية
- تقارير متقدمة وتحليلات
- إدارة الخصومات والعروض
- دعم أنواع دفع متعددة
